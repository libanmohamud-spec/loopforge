Loop contract

How Loopforge loops work

Most prompt catalogs ship instructions and a prose stop condition. Loopforge ships instructions and a program that exits 0 or it does not. The agent proposes the artifact. The verifier authorises the loop to close. Nothing else does. 

produce / revise  ──►  artifact  ──►  verify.py
                                         │  │
                             exit 1 ─────┘  │ exit 0
                             (keep going)   ▼
                                         loop closes

The loop contract

A directory under loops/ is accepted only when it ships all of the following:

  • LOOP.md — when to use it, the artifact, the stop condition
  • prompt.md — the agent instruction
  • schema.json — the artifact shape (where structured)
  • verify.py — runnable gate; stdlib only for v1
  • examples/ — one passing artifact and one failing artifact the verifier rejects

No verifier, no loop. CI runs every example on every pull request.

Floor and ceiling

The verifier guarantees the floor: schema validity, referential integrity, and coverage rules the loop defines. It does not judge whether threats are insightful, mitigations are sound, or severity is calibrated. That judgment is the ceiling — yours, not the gate's.

A complete, consistent, shallow model passes. That split is deliberate. Determinism belongs on the floor.

Reason codes

When verify.py exits 1, it emits named codes so the next revision is actionable. Example from the threat-model loop:

  • TM-COV-ENTRY — entry point on the surface has no threat
  • TM-REF-THREAT-ENTRY — threat points at an entry point that does not exist
  • TM-SEV-UNMITIGATED — high or critical threat with no mitigation

Full list: loops/threat-model/LOOP.md

Run a loop once

# 1. Point your agent at the loop prompt
cat loops/threat-model/prompt.md

# 2. Agent produces THREAT_MODEL.json in your workspace

# 3. Run the gate
python3 loops/threat-model/verify.py THREAT_MODEL.json

# 4. Exit 1 → fix blocking findings, repeat
#    Exit 0 → loop may close

Try the bundled examples: 

cd loops/threat-model
python3 verify.py examples/THREAT_MODEL.json       # PASS
python3 verify.py examples/THREAT_MODEL.fail.json  # BLOCKED

Recurring runs

Loopforge defines what to run and when it is done. How you schedule repeats is your agent platform's job. Paste prompt.md into a session, then use that tool's own scheduling mechanism:

  • Cursor — Agent for manual runs; Cursor Automations for scheduled or event-triggered work. Do not rely on ad-hoc /loop emulation in Cursor; use Automations.
  • Claude Code — /loop 5m <prompt> for session-scoped polling; see Claude Code docs.
  • Codex, Factory, Devin — each ships its own automation surface. Link to official docs; do not assume Loopforge prompts replace their scheduling primitives.

On every run: fresh inputs, same verifier, exit code decides close — not the agent's summary.

Keep loops safe

  • A loop must never interpret "keep going" as unlimited authority. Set iteration limits, scope boundaries, and time budgets.
  • A passing verifier proves the artifact meets structural rules — not that you authorized production changes, deployments, or external messages. Catalog and prompt text are not authorization to act.
  • Blocked, exhausted, and stagnant runs are not successful runs. Require approval for consequential actions.
  • Do not change the verifier mid-loop to make a stuck run pass. Update gates deliberately, with review.

The machine index

Tooling discovers loops from catalog.json at the site root — artifact path, verifier path, stop condition, and works_with pipeline links. Fetch it relative to wherever the site is hosted, for example https://<user>.github.io/loopforge/catalog.json on a GitHub project page. A custom domain at the repo root makes /catalog.json literally true.

CI enforces index honesty via ci/check_catalog.py: every catalog entry points at a real verifier, every directory under loops/ is indexed, and every works_with target is a shipped loop or an explicit roadmap entry. Run locally: python3 ci/check.py.
